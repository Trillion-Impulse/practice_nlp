# NLP란?
- NLP는 컴퓨터가 인간의 언어를 이해하고 처리하도록 만드는 기술
- Natural Language Processing
- ex
    - 사람이 보면 바로 이해하는 문장
        ```
        삼성전자 주가가 오늘 크게 상승했다.
        ```
    - 컴퓨터는 그냥 문자열 데이터로만 봄
        - NLP는 위의 문장을 아래와 같이 바꿈
            ```
            [삼성전자] [주가] [상승]
            ```
        - 문장을 의미 있는 데이터로 변환하는 과정
            - 토큰화, 형태소 분석 등 사용

## NLP는 언제 사용하는가?
- 다양한 텍스트 데이터를 처리하는 경우에 사용
- ex
    - 뉴스
        ```
        삼성전자는 AI 반도체 시장에서 경쟁력을 강화하고 있다.
        ```
    - 쇼핑몰 리뷰
        ```
        배송 빠르고 제품 좋습니다
        ```
    - SNS
        ```
        이 영화 진짜 재밌다
        ```
- 텍스트 데이터는 그대로 분석하기 어렵기 때문에 NLP 처리

## NLP 왜 하는가?
- 여러가지 목적이 존재
    - 키워드 추출
        ```
        삼성전자 AI 반도체 경쟁력 강화
        ↓
        삼성전자
        AI
        반도체
        ```
    - 감성 분석
        ```
        배송 빠르고 제품 좋습니다
        ↓
        긍정
        ```
    - 문서 분류
        ```
        삼성전자 반도체 투자 확대
        ↓
        카테고리: 경제
        ```
    - 요약
        ```
        긴 기사
        ↓
        핵심 문장 요약
        ```

## NLP 처리 단계
- 보통의 순서
    ```
    텍스트 수집
    ↓
    텍스트 정제
    ↓
    토큰화
    ↓
    불용어 제거
    ↓
    벡터화
    ↓
    분석 / 모델
    ```
- 텍스트 수집
- 텍스트 정제
    - 불필요한 문자 제거
        ```
        삼성전자!!! 주가 상승???
        ↓
        삼성전자 주가 상승
        ```
- 토큰화 (Tokenization)
    - 문장을 단어 단위로 나누는 작업
        ```
        삼성전자 주가 상승
        ↓
        ["삼성전자", "주가", "상승"]
        ```
- 불용어 제거
    - 텍스트에서 분석에 크게 영향을 미치지 않는 단어들을 제거하는 과정
        ```
        나는 오늘 학교에 갔다
        ↓
        학교
        갔다

        불용어
        - 나는
        - 오늘
        - 에
        ```
- 벡터화
    - 컴퓨터는 숫자만 이해하므로 단어를 숫자 벡터로 변환
        ```
        삼성전자 → [0.12, 0.88, 0.44 ...]
        ```
    - 여기서 사용하는 기술
        - Word2Vec, BERT, embedding

# 크롤링 데이터를 정제하는 경우
- 크롤링 데이터를 NLP 처리할 경우, 텍스트 정제 단계는 필수적
- 대부분의 NLP 도구들이 기본적인 텍스트 처리 작업을 일부 자동으로 해줌
- 완벽한 정제를 위해 사용자가 직접 처리해야하는 경우 존재
- 텍스트의 특성상 잡음(noise)이 많이 포함되어 있기 때문
- 불용어는 대개 NLP 처리에서 제거됨
    - 불용어는 분석에 큰 도움이 되지 않는 단어들
        - 예: '이', '저', '그'와 같은 대명사나 연결어 등
- 대부분의 NLP 라이브러리들은 기본적인 불용어 목록을 제공하지만, 도메인에 맞는 불용어 리스트를 사용자가 직접 추가해야하는 경우 존재
- 정규화(Normalization)
    - 한국어의 경우 동일 의미의 단어들이 여러 형태로 나타날 수 있음
    - 예
        ```
        밥을 먹다
        밥 먹다
        ```
        - 두 가지는 동일한 의미를 가짐
        - 이런 표현들을 정규화하는 작업이 필요할 수 있음

# 감정 분석 프로젝트

1. 프로젝트 목표
- 문장의 긍정/부정 분류
- 예시
    ```
    이 영화 진짜 재밌다 → 긍정
    완전 시간 낭비였다 → 부정
    ```

2. 관련 기술
    1. pandas
        - 데이터 다루기
        - 예: csv 파일 읽기
    2. scikit-learn
        - 머신러닝 모델
        - TF-IDF
        - 데이터 분리
        - 파이썬 머신러닝 라이브러리 표준
    3. NLTK 또는 KoNLPy
        - 텍스트 처리
        - 토큰화

3. 핵심 NLP 개념
    1. 텍스트 전처리
        - 특수문자 제거
        - 소문자 변환
        - 불용어 제거
        - 토큰화
    2. 토큰화
        - 문장을 단어 단위로 쪼개는 것
        - 예시
            ```
            이 영화 진짜 재밌다
            ["이","영화","진짜","재밌다"]
            ```
    3. 벡터화
        - 컴퓨터는 문자를 이해 못함
        - 그래서 텍스트를 숫자로 변환
        - 대표적인 방법
            - Bag-of-Words
            - TF-IDF
    4. TF-IDF
        - Term Frequency – Inverse Document Frequency
        - Term Frequency = 단어 빈도
            - 특정 문서 안에서 어떤 단어가 얼마나 자주 등장하는지를 의미
            - 많이 나오면 그 문서에서 중요할 가능성이 있다고 봄
        - Inverse Document Frequency = 역문서 빈도
            - 전체 문서들 중에서 그 단어가 얼마나 희귀한지를 나타냄
            - 많은 문서에 공통으로 등장하는 단어는 중요도가 낮아짐
        - 두개를 곱함
            - TF-IDF=TF×IDF
            - 특정 문서에서는 자주 나오지만
            - 다른 문서에서는 잘 안 나오는 단어
            - 그 문서를 특징짓는 핵심 단어
        - TF-IDF = "문서에서 중요한 키워드를 찾는 통계 방법"
        - NLP에서 가장 유명한 방법
        - 중요한 단어에 높은 점수를 줌
        - 예시
            ```
            이 영화 진짜 재밌다
            이 영화 별로다
            ```
            - "영화": 자주 나옴 → 중요도 낮음
            - "재밌다": 특정 문장 → 중요도 높음
        - 수식에 관하여
            1. 시작
                - 문제: 문서를 숫자로 표현하고 싶다
                    - 문서에서 많이 나오는 단어가 중요하지 않을까?
                - 문서들
                    - 문서1: 나는 사과를 먹었다
                    - 문서2: 나는 사과를 좋아한다
                    - 문서3: 나는 바나나를 먹었다
            2. 첫번째 아이디어: TF (Term Frequency)
                - TF=단어가문서에서등장한횟수
            3. 문제: 어떤 단어는 모든 문서에 다 있음
                - ex: 나는, 그리고 등
                - 모든 문서에 있는 단어는 중요도를 낮추자
            4. 두번째 아이디어: IDF (Inverse Document Frequency)
                - 많은 문서에 나타날수록 중요도 낮다
                - IDF는 “전체 문서에서 봤을 때 이 단어가 얼마나 희귀한가”를 측정하려는 값
                - 수식
                    $$
                    중요도 |propto |frac{1}{등장한문서수}
                    $$
                - 예
                    | 단어  | 등장 문서수 | 값 |
                    | --- | ------ | --- |
                    | 나는  | 3      | 1/3 |
                    | 사과  | 2      | 1/2 |
                    | 바나나 | 1      | 1 |
            5. 문제: 값 차이가 너무 작다
                - 문서가 많아지면
                    | 단어          | 등장 문서     |
                    | ----------- | --------- |
                    | the         | 1,000,000 |
                    | transformer | 10        |
                - 곱셈 계산할 때 숫자 차이가 너무 커짐
            6. 로그 사용
                - 로그의 역할: 큰 숫자 차이를 줄여줌
            7. 그래서 IDF 수식이 탄생
                - 수식
                    $$
                    IDF = \log \left(\frac{전체문서수}{단어가등장한문서수}\right)
                    $$
            8. 두 아이디어 합침
                - TF-IDF=TF×IDF
                - 이 문서에서는 자주 나오지만 다른 문서에서는 잘 안 나오는 단어
    5. 머신러닝 분류기
        - 문장 → 긍정 / 부정을 예측해야 함
        - 그래서 모델 사용
            - 대표적으로
                - Naive Bayes classifier
                - Logistic Regression

4. 전체 NLP 파이프라인
- 전체 흐름
    ```
    데이터 수집

    ↓
    TF-IDF 벡터화

    ```
