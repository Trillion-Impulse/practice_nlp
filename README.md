# NLP란?
- NLP는 컴퓨터가 인간의 언어를 이해하고 처리하도록 만드는 기술
- Natural Language Processing
- ex
    - 사람이 보면 바로 이해하는 문장
        ```
        삼성전자 주가가 오늘 크게 상승했다.
        ```
    - 컴퓨터는 그냥 문자열 데이터로만 봄
        - NLP는 위의 문장을 아래와 같이 바꿈
            ```
            [삼성전자] [주가] [상승]
            ```
        - 문장을 의미 있는 데이터로 변환하는 과정
            - 토큰화, 형태소 분석 등 사용

## NLP는 언제 사용하는가?
- 다양한 텍스트 데이터를 처리하는 경우에 사용
- ex
    - 뉴스
        ```
        삼성전자는 AI 반도체 시장에서 경쟁력을 강화하고 있다.
        ```
    - 쇼핑몰 리뷰
        ```
        배송 빠르고 제품 좋습니다
        ```
    - SNS
        ```
        이 영화 진짜 재밌다
        ```
- 텍스트 데이터는 그대로 분석하기 어렵기 때문에 NLP 처리

## NLP 왜 하는가?
- 여러가지 목적이 존재
    - 키워드 추출
        ```
        삼성전자 AI 반도체 경쟁력 강화
        ↓
        삼성전자
        AI
        반도체
        ```
    - 감성 분석
        ```
        배송 빠르고 제품 좋습니다
        ↓
        긍정
        ```
    - 문서 분류
        ```
        삼성전자 반도체 투자 확대
        ↓
        카테고리: 경제
        ```
    - 요약
        ```
        긴 기사
        ↓
        핵심 문장 요약
        ```

## NLP 처리 단계
- 보통의 순서
    ```
    텍스트 수집
    ↓
    텍스트 정제
    ↓
    토큰화
    ↓
    불용어 제거
    ↓
    벡터화
    ↓
    분석 / 모델
    ```
- 텍스트 수집
- 텍스트 정제
    - 불필요한 문자 제거
        ```
        삼성전자!!! 주가 상승???
        ↓
        삼성전자 주가 상승
        ```
- 토큰화 (Tokenization)
    - 문장을 단어 단위로 나누는 작업
        ```
        삼성전자 주가 상승
        ↓
        ["삼성전자", "주가", "상승"]
        ```
- 불용어 제거
    - 텍스트에서 분석에 크게 영향을 미치지 않는 단어들을 제거하는 과정
        ```
        나는 오늘 학교에 갔다
        ↓
        학교
        갔다

        불용어
        - 나는
        - 오늘
        - 에
        ```
- 벡터화
    - 컴퓨터는 숫자만 이해하므로 단어를 숫자 벡터로 변환
        ```
        삼성전자 → [0.12, 0.88, 0.44 ...]
        ```
    - 여기서 사용하는 기술
        - Word2Vec, BERT, embedding

# 크롤링 데이터를 정제하는 경우
- 크롤링 데이터를 NLP 처리할 경우, 텍스트 정제 단계는 필수적
- 대부분의 NLP 도구들이 기본적인 텍스트 처리 작업을 일부 자동으로 해줌
- 완벽한 정제를 위해 사용자가 직접 처리해야하는 경우 존재
- 텍스트의 특성상 잡음(noise)이 많이 포함되어 있기 때문
- 불용어는 대개 NLP 처리에서 제거됨
    - 불용어는 분석에 큰 도움이 되지 않는 단어들
        - 예: '이', '저', '그'와 같은 대명사나 연결어 등
- 대부분의 NLP 라이브러리들은 기본적인 불용어 목록을 제공하지만, 도메인에 맞는 불용어 리스트를 사용자가 직접 추가해야하는 경우 존재
- 정규화(Normalization)
    - 한국어의 경우 동일 의미의 단어들이 여러 형태로 나타날 수 있음
    - 예
        ```
        밥을 먹다
        밥 먹다
        ```
        - 두 가지는 동일한 의미를 가짐
        - 이런 표현들을 정규화하는 작업이 필요할 수 있음

# 감정 분석 프로젝트

1. 프로젝트 목표
- 문장의 긍정/부정 분류
- 예시
    ```
    이 영화 진짜 재밌다 → 긍정
    완전 시간 낭비였다 → 부정
    ```

2. 관련 기술
    1. pandas
        - 데이터 다루기
        - 예: csv 파일 읽기
    2. scikit-learn
        - 머신러닝 모델
        - TF-IDF
        - 데이터 분리
        - 파이썬 머신러닝 라이브러리 표준
    3. NLTK 또는 KoNLPy
        - 텍스트 처리
        - 토큰화

3. 핵심 NLP 개념
    1. 텍스트 전처리
        - 특수문자 제거
        - 소문자 변환
        - 불용어 제거
        - 토큰화
    2. 토큰화
        - 문장을 단어 단위로 쪼개는 것
        - 예시
            ```
            이 영화 진짜 재밌다
            ["이","영화","진짜","재밌다"]
            ```
    3. 벡터화
        - 컴퓨터는 문자를 이해 못함
        - 그래서 텍스트를 숫자로 변환
        - 대표적인 방법
            - Bag-of-Words
            - TF-IDF
    4. TF-IDF
        - Term Frequency – Inverse Document Frequency
        - Term Frequency = 단어 빈도
            - 특정 문서 안에서 어떤 단어가 얼마나 자주 등장하는지를 의미
            - 많이 나오면 그 문서에서 중요할 가능성이 있다고 봄
        - Inverse Document Frequency = 역문서 빈도
            - 전체 문서들 중에서 그 단어가 얼마나 희귀한지를 나타냄
            - 많은 문서에 공통으로 등장하는 단어는 중요도가 낮아짐
        - 두개를 곱함
            - TF-IDF=TF×IDF
            - 특정 문서에서는 자주 나오지만
            - 다른 문서에서는 잘 안 나오는 단어
            - 그 문서를 특징짓는 핵심 단어
        - TF-IDF = "문서에서 중요한 키워드를 찾는 통계 방법"
        - NLP에서 가장 유명한 방법
        - 중요한 단어에 높은 점수를 줌
        - 예시
            ```
            이 영화 진짜 재밌다
            이 영화 별로다
            ```
            - "영화": 자주 나옴 → 중요도 낮음
            - "재밌다": 특정 문장 → 중요도 높음
        - 수식에 관하여
            1. 시작
                - 문제: 문서를 숫자로 표현하고 싶다
                    - 문서에서 많이 나오는 단어가 중요하지 않을까?
                - 문서들
                    - 문서1: 나는 사과를 먹었다
                    - 문서2: 나는 사과를 좋아한다
                    - 문서3: 나는 바나나를 먹었다
            2. 첫번째 아이디어: TF (Term Frequency)
                - TF=단어가문서에서등장한횟수
                - 수식  
                    ![TF](https://latex.codecogs.com/png.image?\dpi{120}TF=\frac{t}{d})
                    - t: 해당 문서에서 단어 t가 등장한 횟수
                    - d: 문서 d의 전체 단어 수
            3. 문제: 어떤 단어는 모든 문서에 다 있음
                - ex: 나는, 그리고 등
                - 모든 문서에 있는 단어는 중요도를 낮추자
            4. 두번째 아이디어: IDF (Inverse Document Frequency)
                - 많은 문서에 나타날수록 중요도 낮다
                - IDF는 “전체 문서에서 봤을 때 이 단어가 얼마나 희귀한가”를 측정하려는 값
                - 수식  
                    ![Importance](https://latex.codecogs.com/png.image?\dpi{120}Importance\propto\frac{1}{document})
                    - document: 등장한 문서 수
                - 예
                    | 단어  | 등장 문서수 | 값 |
                    | --- | ------ | --- |
                    | 나는  | 3      | 1/3 |
                    | 사과  | 2      | 1/2 |
                    | 바나나 | 1      | 1 |
            5. 문제: 값 차이가 너무 작다
                - 문서가 많아지면
                    | 단어          | 등장 문서     |
                    | ----------- | --------- |
                    | the         | 1,000,000 |
                    | transformer | 10        |
                - 곱셈 계산할 때 숫자 차이가 너무 커짐
            6. 로그 사용
                - 로그의 역할: 큰 숫자 차이를 줄여줌
            7. 그래서 IDF 수식이 탄생
                - 수식  
                    ![idf](https://latex.codecogs.com/png.image?\dpi{120}IDF=\log\left(\frac{N}{df}\right))
                    - N: 전체 문서 수
                    - df: 단어가 등장한 문서 수
            8. 두 아이디어 합침
                - TF-IDF=TF×IDF
                - 이 문서에서는 자주 나오지만 다른 문서에서는 잘 안 나오는 단어
    5. 머신러닝 분류기
        - 문장 → 긍정 / 부정을 예측해야 함
        - 그래서 모델 사용
            - 대표적으로
                - Naive Bayes classifier
                - Logistic Regression

4. 전체 NLP 파이프라인
- 전체 흐름
    ```
    데이터 수집
    ↓
    전처리

    TF-IDF 벡터화
    ↓
    모델 학습

    ```

# 데이터셋
- 머신러닝 모델이 학습하고 평가하는 자료 모음
- 데이터셋이 많고 다양할수록 모델 성능이 좋아질 가능성 ↑
- 데이터 품질이 나쁘면, 아무리 좋은 모델도 성능이 안 나옴
- ML(머신러닝)은 지도학습 방식이므로 정답이 함께 포함되어있음
    - 지도학습: 정답이 있는 데이터로 학습하는 방법
        ```
        입력 + 정답
        ↓
        패턴 학습
        ↓
        예측
        ```

## 데이터셋의 기본 구성
- ML(머신러닝)에서는 보통 데이터를 입력(X)과 정답(Y)으로 나눔
- 표
    | 구성           | 설명          | 감정분석 예시    |
    | ------------ | ----------- | ---------- |
    | X (features) | 모델이 학습할 데이터 | 문장, 리뷰 텍스트 |
    | Y (labels)   | 모델이 예측할 정답  | 긍정 / 부정    |

## 데이터셋의 구분
- 표
    | 종류                     | 용도                      |
    | ---------------------- | ----------------------- |
    | **훈련(Training) 데이터**   | 모델을 학습시키는 데 사용          |
    | **검증(Validation) 데이터** | 학습 중 하이퍼파라미터 조정, 과적합 방지 |
    | **테스트(Test) 데이터**      | 학습이 끝난 모델 평가            |
- 한 번도 본 적 없는 데이터로 테스트해야 모델이 실제로 잘 작동하는지 알 수 있음
- 훈련 데이터
    - 모델을 학습시키는 용도
    - 전체 데이터의 약 60%~80%
    - 모델이 패턴을 배우는 용도
- 검증 데이터
    - 학습 중 모델 성능 평가
    - 하이퍼파라미터 튜닝, 모델 선택
    - 약 10%~20%
    - 모델의 과적합 방지
        - 과적합: 모델이 데이터를 외워버리는 현상
        - 검증 데이터를 통해 모델이 새로운 데이터를 얼마나 잘 예측하는지 검증
- 테스트 데이터
    - 최종 모델 성능 평가
    - 학습/튜닝에는 절대 사용 X
    - 약 10%~20%
- 실무에서는 train, validate, test 비율을 프로젝트 상황에 맞게 조정

## 데이터 분리 방법
- Python에서는 scikit-learn의 train_test_split 사용
    ```
    from sklearn.model_selection import train_test_split

    # 데이터 분리: 학습 70%, 테스트 30%
    X_train, X_test, y_train, y_test = train_test_split(X, Y, test_size=0.3, random_state=42)

    # 학습에서 일부를 검증으로 분리: 학습 80%, 검증 20%
    X_train, X_val, y_train, y_val = train_test_split(X_train, y_train, test_size=0.2, random_state=42)
    ```

## 데이터셋 구성 체크리스트
- 표
    | 체크                  | 이유                    |
    | ------------------- | --------------------- |
    | train/val/test 분리   | 과적합 방지, 성능 객관적 평가     |
    | 클래스 균형 확인           | 불균형 데이터 시 f1-score 사용 |
    | 전처리 일관성             | train/val/test 동일 처리  |
    | 시드 고정(random_state) | 실험 재현성 확보             |
- 시드 고정
    - 시드(seed)란?
        - 컴퓨터는 완전히 랜덤한 수를 만들지 못함
        - 대신 의사 난수(Pseudo-Random Number)를 생성
            - 랜덤 함수: 매번 다른 숫자 생성
            - 의사 난수: 내부적으로 시작점(seed)을 기준으로 숫자 생성 → 같은 seed면 같은 랜덤 값
            - seed를 고정하면 랜덤한 과정도 항상 동일하게 반복 됨
        - 랜덤 난수 생성 초기값
    - ML에서 시드를 고정하는 이유
        - ML에서 랜덤이 쓰이는 경우가 많음
        - 데이터셋 분리에서 train_test_split의 random_state 파라미터
            - random_state 고정하면 항상 같은 tain/test 데이터로 분리
        - 모델 초기화
            - 신경망 가중치 초기화도 랜덤 → seed 고정하면 모델 초기 상태 동일
        - 배깅/부트스트랩
            - 앙상블, 랜덤포레스트 등에서 샘플링이 랜덤 → seed 고정하면 결과 동일
        - 실험 재현성 확보
        - 버그 추적 용이
        - 데이터 분리, 모델 초기화, 샘플링 등에 적용

# 전처리
- 원본 텍스트 데이터를 분석이나 모델 학습에 적합한 형태로 가공하는 과정
- 불필요한 요소 제거, 형태 정리, 토큰 분리 등을 수행
- 전처리가 중요한 이유
    - 노이즈 제거 → 데이터 품질 향상
    - 데이터 일관성 확보
    - 모델 학습 효율 증가
    - 성능 향상

## 텍스트 정규화
- Text Normalization
- 텍스트를 일관된 형태로 변환하는 과정
- 예
    - 대문자 → 소문자
    - 줄임말 변환
    - 반복 문자 축소

## 특수문자 및 불필요한 문자 제거
- 모델 학습에 필요 없는 기호를 제거
- 제거 대상 예
    - 특수문자
    - HTML 태그
    - URL
    - 이모지

## 토큰화
- Tokenization
- 문장을 단어 또는 의미 단위로 분리하는 작업
- 예
    ```
    I love NLP
    ↓ 토큰화
    ["I", "love", "NLP"]
    ```

## 불용어 제거
- Stopword Removal
- 분석에 큰 의미가 없는 단어를 제거
- 예
    ```
    I love studying NLP
    ↓ 불용어 제거
    ["love", "studying", "NLP"]
    ```

## 어간 추출
- Stemming
- 단어의 어간만 남기는 과정
- 예
    ```
    playing
    played
    plays
    ↓ 어간 추출
    play
    ```

## 표제어 추출
- Lemmatization
- 단어를 사전에 있는 기본형으로 변환
- 예
    ```
    am, are, is
    → be
    ```
- 어간 추출보다 의미적으로 정확

## 형태소 분석
- Morphological Analysis
- 문장을 형태소 단위로 분해하고 품사를 분석
- 예
    ```
    나는 학교에 간다
    ↓ 형태소 분석
    나 / 는 / 학교 / 에 / 가 / ㄴ다
    ```
- 한국어 NLP에서 매우 중요한 단계

## 벡터화
- 텍스트를 숫자 벡터로 변환
- 모델은 숫자만 처리할 수 있기 때문
- 대표적인 방법들
    - Bag of Words
    - TF-IDF
    - Word2Vec
    - FastText
    - BERT embedding

### vectorizer
- vectorizer는 벡터화 변환기
- vectorizer에 데이터를 fit하면 학습된 모델 객체가 됨
    - 내부에 단어 사전, 각 단어의 중요도 계산 방식을 저장
- 이 학습된 vectorizer를 저장해두고 나중에 그대로 사용
- joblib 라이브러리
    - dump를 이용해 메모리에 저장되어있는 vectorizer를 파일로 저장
        - dump는 load의 반대라 생각하면 됨

### fit
- 데이터 전체를 보고 통계를 계산하는 과정
- 데이터의 패턴(기준)을 학습
- 실제로 데이터를 바꾸지는 않음
- 어떻게 변환할지 기준을 정하는 단계
- 주의
    - 테스트 데이터에 fit 하면 안됨
    - 테스트 데이터 정보를 미리 보면 데이터 누수 발생
        - 데이터 누수: 모델이 미래 데이터를 미리 본 상태  

### transform
- 배운 기준으로 실제 데이터를 바꾸는 단계
- 이미 만들어진 기준으로 변환만 수행
- fit에서 학습한 기준을 사용해서 데이터를 변환

### fit_transform
- fit + transform을 한 번에 수행
- fit이 포함되어 있기 때문에 훈련 데이터에서만 사용

## pandas
- 데이터 전처리에서 가장 많이 사용하는 Python 라이브러리
- 표 형태 데이터(DataFrame)를 다루기 쉽게 해줌
- 텍스트 데이터 전처리에서도 핵심적으로 사용됨

### pandas 사용 이유
- 대량의 텍스트 데이터를 효율적으로 처리
- 열(column) 단위로 데이터 관리 가능
- 다양한 전처리 작업을 간단하게 수행 가능

### Series
- 1차원 데이터 구조
- 리스트처럼 보이지만, 인덱스(index)가 붙어 있음
- 하나의 열(column) 데이터라고 생각하면 쉬움
- 딕셔너리처럼 사용 가능

### DataFrame
- 2차원 데이터 구조 (행 + 열)
- 엑셀 표(table)처럼 생김
- 여러 개의 Series가 모여서 만들어짐

### Series와 DataFrame
- 표
    | 구분 | Series    | DataFrame   |
    | -- | --------- | ----------- |
    | 차원 | 1차원       | 2차원         |
    | 구조 | 단일 열      | 여러 열        |
    | 비유 | 리스트 + 인덱스 | 엑셀 표        |
    | 구성 | 값들의 집합    | Series들의 집합 |
- 참고
    ```
    type(df['A'])
    ```
    - 결과는 Series
    - DataFrame에서 한 열을 선택하면 Series가 됨

### apply
- 각 데이터(문장)에 동일한 전처리 함수를 적용할 때 사용
- 각 행이나 열에 특정 함수를 적용할 때 사용
- 기본 구조
    ```
    DataFrame.apply(func, axis=0)
    ```
    - func: 적용할 함수
    - axis: 적용 방향(0: 열(기본값), 1: 행)
- 예
    ```
    df["cleaned_text"] = df["text"].apply(clean_text)
    ```
- apply vs map vs applymap
    | 함수         | 적용 대상              | 특징        |
    | ---------- | ------------------ | --------- |
    | `apply`    | DataFrame / Series | 행 또는 열 단위 |
    | `map`      | Series             | 요소 하나씩    |
    | `applymap` | DataFrame          | 전체 요소 각각  |
- 주의사항
    - apply는 반복문 기반이라 속도가 느릴 수 있음
    - 가능하면 벡터화 연산 사용 권장
        ```
        # 느림
        df['A'].apply(lambda x: x * 2)

        # 빠름 (권장)
        df['A'] * 2
        ```
- 언제 사용하는가?
    - 복잡한 로직을 각 행/열에 적용할 때
    - 기존 벡터 연산으로 처리하기 어려운 경우
    - 사용자 정의 함수가 필요한 경우
